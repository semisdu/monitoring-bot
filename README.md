# 🤖 Monitoring Bot

Универсальная система мониторинга инфраструктуры с поддержкой Docker, PVE, PBS и мультиязычностью.

## 📋 Возможности

- ✅ Мониторинг Docker контейнеров (алерты при падении)
- ✅ Мониторинг логов внутри контейнеров (ошибки, exceptions)
- ✅ Мониторинг Proxmox VE (статус VM, CPU, RAM)
- ✅ Мониторинг бэкапов PBS (возраст, свободное место)
- ✅ Проверка доступности сайтов (HTTP/HTTPS)
- ✅ Ежедневные отчёты в Telegram
- ✅ Мультиязычность (🇷🇺 Русский, 🇺🇦 Українська, 🇬🇧 English)
- ✅ Универсальная архитектура - добавляй любые серверы без изменения кода

## 📖 История создания

### 🗓️ 2 января 2026 года

К тому моменту мы уже месяц тестировали один образовательный сайт. Каждое утро происходило одно и то же: сайт ложился в хлам.

### 🌙 Ночной кошмар

А причина была проста:

- Ночью PBS (Proxmox Backup Server) начинал свою работу
- Бэкапил все виртуальные машины
- Сайт на VM притормаживал, подвисал, а иногда и падал
- К утру мы получали упавшую базу и кучу вопросов

### 💡 Момент прозрения

Я сидел удалённо, смотрел на очередной упавший сайт и понял:

> "Я не могу каждый раз вручную проверять, что пошло не так. Нужен инструмент, который будет делать это за меня".

### 🔨 Как рос проект

Сначала были простые статические скрипты. Потом — первые кнопки в Telegram, которые просто отправляли команды.

Но чем больше становилось серверов и контейнеров, тем сильнее росло понимание:

> Статические кнопки — это тупик.

Так родилась идея **динамического меню**, которое само подстраивается под то, что реально есть в системе:

- Добавил новый сервер → кнопка появилась сама
- Убрал контейнер → исчезла из меню
- Не нужно переписывать код, просто правлю конфиг

### ⚡ Пауза в разработке

Полтора месяца проект просто лежал. Не потому, что не хотелось писать — просто не было света. Ситуация, на которую я не мог повлиять.

Но как только появилась возможность — я вернулся к коду. И доделал до конца.

### 🎯 Релиз

**06.03.2026** — публичный релиз.

Из желания просто понять, почему падает сайт по ночам, выросла полноценная система мониторинга, которая:

- ✅ Следит за всем сама
- ✅ Адаптируется под инфраструктуру
- ✅ Не требует правки кода при добавлении серверов
- ✅ Даёт мне спать по ночам 😴

## 🔑 Настройка SSH-доступа (ВАЖНО!)

Для мониторинга серверов боту нужен доступ по SSH. **Каждый, кто разворачивает бота, должен использовать СВОИ собственные SSH-ключи**.

### 1. Создайте свою пару SSH-ключей (если ещё нет)

`ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519_monitoring`

> Можно использовать любое имя файла, главное - указать его в конфиге.

### 2. Скопируйте публичный ключ на все сервера, которые хотите мониторить

Для каждого сервера выполните:

`ssh-copy-id -i ~/.ssh/id_ed25519_monitoring.pub username@server-ip`

Вас попросят ввести пароль пользователя на сервере. После этого ключ будет добавлен в `~/.ssh/authorized_keys` на удалённом сервере.

### 3. Укажите имя ключа в конфиге

В файле `config/config.yml` для каждого сервера укажите **только имя файла** ключа:

`servers:`
`  - id: "my-server"`
`    user: "username"`
`    ssh_key: "id_ed25519_monitoring"  # только имя файла, без пути!`

### 4. Укажите директорию с ключами (опционально)

В секции `paths` можно указать, где искать ключи:

`paths:`
`  ssh_keys: "~/.ssh"  # по умолчанию ищет в ~/.ssh/`

Если не указывать, бот сам найдёт ключ в:
- Указанной в конфиге директории
- Стандартной `~/.ssh/`
- Текущей директории проекта
- Папке `keys/` в проекте

### 5. Проверьте подключение

Убедитесь, что бот сможет подключиться:

`ssh -i ~/.ssh/id_ed25519_monitoring username@server-ip`

Если вы зашли без запроса пароля — всё готово к запуску бота! 🚀

## 📁 Структура конфига

`telegram:`
`  token: "YOUR_BOT_TOKEN"  # Получить у @BotFather`
`  admin_chat_id: 123456789  # Ваш Telegram ID`

`servers:`
`  - id: "my-server"        # Уникальный ID`
`    name: "Мой сервер"      # Отображаемое имя`
`    host: "192.168.1.100"   # IP или домен`
`    user: "username"        # Пользователь SSH`
`    ssh_key: "id_ed25519"   # Имя ключа (бот найдёт сам)`
`    docker_enabled: true    # Есть Docker?`
`    containers:              # Список контейнеров`
`      - name: "postgres"`
`        critical: true`

## 🐳 Мониторинг Docker

Система отслеживает:

- 🔴 Падение критических контейнеров
- 📝 Ошибки в логах внутри контейнеров
- 🔄 Статус всех контейнеров

## 🏗 Мониторинг Proxmox (PVE)

- 📊 Статус виртуальных машин
- 📈 Нагрузка CPU/RAM
- ⚠️ Алерты при превышении порогов

## 💾 Мониторинг бэкапов (PBS)

- 📅 Возраст последнего бэкапа
- 💽 Свободное место в хранилище
- ⏰ Предупреждения об устаревании

## 🤖 Команды Telegram

- `/start` - Главное меню
- `/help` - Справка
- `/status` - Статус серверов
- `/sites` - Проверка сайтов
- `/docker` - Управление Docker
- `/alerts` - Активные алерты
- `/logs` - Просмотр логов
- `/stats` - Статистика
- `/pve` - Статус Proxmox
- `/pbs` - Статус бэкапов
- `/report` - Отчёты и аналитика
- `/trends` - Тренды ошибок

## 🔧 Требования

- Python 3.8+
- SSH доступ к серверам (свои ключи!)
- Telegram Bot Token
- Для PVE: доступ к API (root@pam)
- Для PBS: доступ к API (root@pbs)
